# Examples Pages Design

**Date:** 2026-04-24
**Status:** Approved

## Overview

Add an `examples.mdx` page to each of the 10 domain docs. Each page contains one realistic end-to-end walkthrough showing all phases used in sequence with example commands and sample output snippets.

## Scenarios

| Domain | Scenario |
|--------|----------|
| Design | Designing a DatePicker component |
| Research | Researching user onboarding friction |
| Marketing | Launching a SaaS product |
| Branding | Building a brand for a fintech startup |
| Project Management | Running a mobile app rebuild sprint |
| Legal | Setting up legal for a new SaaS product |
| Coaching | Reviewing a landing page across disciplines |
| Leadership | Building a new engineering team |
| Accessibility | Auditing an e-commerce checkout flow |
| Development | Building a payment integration |

## Page Structure

Each page follows: Scenario context → Per-phase walkthrough (command + description + sample output snippet) → What You End Up With summary.

## Sidebar

Add `"---Examples---", "examples"` to each domain's meta.json after the Memory section.
