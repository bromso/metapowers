# Build a Feature

A structured feature development workflow from research through shipping.

## Steps

1. `/research:discover <feature>` — Understand the problem space
2. `/research:define <feature>` — Define the problem clearly
3. `/development:plan <feature>` — Break into implementation tasks
4. `/development:build <feature>` — Execute with TDD discipline
5. `/development:test <feature>` — Validate test coverage
6. `/accessibility:checklist <feature>` — Quick accessibility check
7. `/coaching:code <feature>` — Get engineering feedback
8. `/development:review <feature>` — Pre-merge code review
9. `/development:ship <feature>` — Merge, PR, or cleanup

## Tips
- For simple features, skip research and start at step 3
- Use `/coaching:ux` if the feature has a UI component
- Run `/accessibility:scope` for a full audit on critical features
