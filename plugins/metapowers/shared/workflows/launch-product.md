# Launch a Product

A complete product launch workflow from market research through go-to-market.

## Steps

1. `/research:discover <product>` — Explore the problem space and market opportunity
2. `/research:define <product>` — Synthesize into a clear problem statement
3. `/research:design <product>` — Generate and evaluate solution concepts
4. `/research:deliver <product>` — Validate the top solution with evidence
5. `/branding:discover <brand>` — Research brand landscape and audience
6. `/branding:strategy <brand>` — Define positioning, values, personality
7. `/branding:verbal <brand>` — Develop voice, tone, and messaging
8. `/branding:visual <brand>` — Define colors, typography, logo usage
9. `/branding:guidelines <brand>` — Package into brand manual
10. `/design:empathize <component>` — Research user needs for key UI
11. `/design:define <component>` — Create component contract
12. `/design:ideate <component>` — Explore design approaches
13. `/development:plan <feature>` — Break the build into tasks
14. `/development:build <feature>` — Execute with TDD discipline
15. `/development:test <feature>` — Validate test coverage
16. `/accessibility:scope <target>` — Define accessibility audit scope
17. `/accessibility:evaluate <target>` — Run automated and manual checks
18. `/accessibility:remediate <target>` — Fix accessibility issues
19. `/accessibility:retest <target>` — Verify fixes and conformance
20. `/marketing:strategy <campaign>` — Plan the go-to-market
21. `/branding:audit <campaign>` — Check brand compliance
22. `/coaching:strategy <launch-plan>` — Get strategic feedback
23. `/metapowers:report` — Cross-domain summary

## Tips
- Replace `<product>`, `<brand>`, `<component>`, etc. with your actual project names
- You don't have to complete every step — skip what doesn't apply
- Steps can span days or weeks — this is a roadmap, not a sprint
