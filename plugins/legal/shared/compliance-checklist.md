# Cross-Regulation Compliance Checklist

A consolidated checklist covering major regulatory frameworks. Use this to assess compliance posture and track implementation progress.

---

## GDPR (General Data Protection Regulation)

- [ ] Lawful basis documented for each processing activity (Art. 6)
- [ ] Consent mechanisms implemented where consent is the lawful basis (Art. 7)
- [ ] Privacy notices provided at point of collection (Art. 13) and for indirect collection (Art. 14)
- [ ] Data mapping complete — all personal data flows documented
- [ ] Data Protection Impact Assessments (DPIA) conducted for high-risk processing (Art. 35)
- [ ] Data Protection Officer (DPO) appointed, if required (Art. 37)
- [ ] Breach notification process established — supervisory authority within 72 hours (Art. 33), data subjects without undue delay (Art. 34)
- [ ] Cross-border transfer mechanisms in place — Standard Contractual Clauses (SCCs), adequacy decisions, or binding corporate rules (Art. 46)
- [ ] Data subject rights procedures implemented — access, rectification, erasure, restriction, portability, objection (Arts. 15-22)
- [ ] Records of processing activities maintained (Art. 30)
- [ ] Data retention policy defined and enforced with documented retention periods

## CCPA / CPRA (California Consumer Privacy Act / California Privacy Rights Act)

- [ ] Notice at collection provided — categories of personal information and purposes (Cal. Civ. Code 1798.100)
- [ ] Opt-out mechanism implemented — "Do Not Sell or Share My Personal Information" link (Cal. Civ. Code 1798.120)
- [ ] Deletion request process established with verification procedures (Cal. Civ. Code 1798.105)
- [ ] Data portability — ability to provide personal information in portable format (Cal. Civ. Code 1798.130)
- [ ] Non-discrimination policy — consumers not penalized for exercising rights (Cal. Civ. Code 1798.125)
- [ ] Service provider agreements updated with CCPA/CPRA-required provisions
- [ ] Financial incentive disclosures provided for loyalty programs and data-for-discount offers (Cal. Civ. Code 1798.125)

## SOC 2 (Service Organization Control 2)

### Security (Common Criteria)

- [ ] Security policies documented and communicated to all personnel
- [ ] Access controls implemented — role-based access, least privilege, MFA
- [ ] Change management process established — development, testing, approval, deployment
- [ ] Incident response plan documented with defined roles, escalation paths, and communication templates
- [ ] Vendor management program — third-party risk assessments, contractual security requirements
- [ ] Encryption in transit (TLS 1.2+) and at rest (AES-256 or equivalent)
- [ ] Monitoring and alerting — security event logging, anomaly detection, regular log review

### Availability

- [ ] Uptime commitments defined and monitored
- [ ] Disaster recovery plan documented and tested
- [ ] Backup procedures implemented and verified

### Processing Integrity

- [ ] Data processing accuracy controls in place
- [ ] Quality assurance procedures documented

### Confidentiality

- [ ] Confidential information classification scheme established
- [ ] Data handling procedures documented for each classification level

### Privacy

- [ ] Privacy notice aligned with SOC 2 privacy criteria
- [ ] Personal information handling procedures documented

## ADA / EAA (Americans with Disabilities Act / European Accessibility Act)

- [ ] WCAG 2.1 Level AA compliance achieved across all public-facing digital properties
- [ ] Accessibility statement published on website with contact information for accessibility issues
- [ ] Alternative formats available — text alternatives for images, captions for video, transcripts for audio
- [ ] Accommodation process documented — clear procedure for requesting and providing reasonable accommodations

## Usage Notes

### Priority Assessment

For each checklist item, assess:

1. **Applicability** — Does this requirement apply to your organization?
2. **Current State** — Is this fully implemented, partially implemented, or not started?
3. **Gap Severity** — What is the risk of non-compliance (regulatory fines, litigation, reputational harm)?
4. **Remediation Timeline** — When must this be addressed?

### Review Schedule

| Regulation | Review Frequency | Trigger Events |
|---|---|---|
| GDPR | Quarterly | New processing activities, data breaches, regulatory guidance |
| CCPA/CPRA | Quarterly | New data collection, regulatory amendments, consumer complaints |
| SOC 2 | Annually (audit cycle) | System changes, incidents, control failures |
| ADA/EAA | Semi-annually | Site redesigns, new features, user complaints |
