## Purpose

Work through a team conflict by understanding perspectives, finding root causes, and planning a constructive resolution conversation.

## Usage

```
/leadership:conflict <situation>
```

**Example:**
```
/leadership:conflict engineers disagree on architecture approach
```

## What It Does

1. Maps each party's perspective and underlying concerns
2. Identifies root cause (roles, priorities, communication, values, or resources)
3. Develops resolution options that address core concerns
4. Plans the facilitation conversation with ground rules
5. Writes to `.metapowers/leadership/<topic>/conflict-resolution.md`
