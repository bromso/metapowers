## Purpose

The Build phase structures your team for clarity and autonomy — who owns what, who decides what, and how work flows between people.

## Usage

```
/leadership:build <team-name>
```

## Prerequisites

Requires `.metapowers/leadership/<team>/02-vision.md`. Use `--skip-checks` to bypass.

## What It Does

1. Maps each team member's role and primary responsibilities
2. Defines ownership areas and decision rights (RAPID framework)
3. Creates a delegation framework matched to member maturity
4. Establishes collaboration norms and escalation paths
5. Writes to `.metapowers/leadership/<team>/03-build.md`

## Output

Creates `.metapowers/leadership/<team>/03-build.md` containing team structure, decision rights, delegation framework, and collaboration norms.
