## Purpose

Structure delegation with clear ownership, authority boundaries, check-in cadence, and done criteria.

## Usage

```
/leadership:delegate <task>
```

**Example:**
```
/leadership:delegate API migration to backend team
```

## What It Does

1. Defines the deliverable and why it matters
2. Matches to the right person based on skills and growth opportunity
3. Sets authority level, support resources, check-in schedule, and done criteria
4. Writes to `.leadership/<topic>/delegation.md`
