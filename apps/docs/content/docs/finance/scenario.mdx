## Purpose

The Scenario utility builds multiple financial scenarios — best, base, and worst case — to understand the range of possible outcomes and prepare contingency plans.

## Usage

```
/finance:scenario <project>
```

## What It Does

1. Defines scenario assumptions for each case (growth rate, churn, pricing)
2. Runs the financial model under each set of assumptions
3. Compares outcomes across scenarios
4. Identifies key decision points and trigger thresholds
5. Documents contingency actions for downside scenarios

## Output

- Three-scenario comparison table (best, base, worst)
- Key metric ranges across scenarios
- Decision triggers and contingency plans
