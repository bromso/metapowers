## Purpose

Utility skills work across all Scrum phases. They have no prerequisites and can be run anytime — for risk management, decision tracking, meeting facilitation, status reporting, or capacity planning.

## Skills

### Risk Assessment

```
/pm:risk-assessment <project>
```

Identifies project risks, assesses probability and impact, and defines mitigation strategies. Maintains a living risk register.

**Output:** Risk register, probability-impact matrix, mitigation plans → `.project/<project>/risk-assessment.md`

### Decision Log

```
/pm:decision-log <project>
```

Records key project decisions with context, alternatives considered, rationale, and stakeholders involved. Provides an audit trail.

**Output:** Decision record, alternatives analysis, rationale documentation → `.project/<project>/decision-log.md`

### Meeting Facilitator

```
/pm:meeting-facilitator <project>
```

Structures and facilitates project meetings — generates agendas, time-boxes discussion topics, captures action items, and produces meeting notes.

**Output:** Meeting agenda, facilitation guide, action items template → `.project/<project>/meeting-facilitator.md`

### Status Report

```
/pm:status-report <project>
```

Generates a project status report with RAG indicators, milestone progress, risks, and upcoming activities. Tailored for different audiences.

**Output:** Status report with RAG status, milestone tracking, risk summary → `.project/<project>/status-report.md`

### Capacity Planning

```
/pm:capacity-planning <project>
```

Plans team capacity across sprints — accounts for availability, planned leave, skill distribution, and sustainable pace.

**Output:** Capacity plan, availability matrix, sustainable pace recommendations → `.project/<project>/capacity-planning.md`
