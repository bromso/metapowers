## Phase Artifacts

| Phase | File | What It Contains |
|-------|------|-----------------|
| Discover | `01-discover.md` | Source bibliography with key takeaways, user and stakeholder insights (needs, pain points, motivations), market and competitive landscape, key themes and patterns, surprises and contradictions, open questions |
| Define | `02-define.md` | Problem statement, "How Might We" questions (3-5), success criteria with metrics, scope definition (in/out), research methods, stakeholder input needed |
| Design | `03-design.md` | 3-5 solution concepts with rationale, evaluation matrix (feasibility, impact, effort, risk per concept), recommended approach, validation plan |
| Deliver | `04-deliver.md` | Executive summary, validation findings (evidence for/against), success criteria assessment, final recommendation, prioritized action items, risks and mitigations, research journey summary |

A `skip-log.md` file is also created if any phase prerequisites are bypassed.

## Artifact Format

All artifacts are Markdown. Research findings use tables for structured comparisons, bullet lists for insights, and blockquotes for direct source quotes.

## Key Artifact Types

- **Source bibliographies** — Curated list of articles, studies, and reports with key takeaways
- **Stakeholder insight maps** — Needs, pain points, and motivations organized by stakeholder group
- **Problem statements** — Clear, specific, actionable problem definitions (solution-free language)
- **HMW questions** — "How Might We" questions that reframe problems as opportunities
- **Solution evaluations** — Feasibility/impact/effort/risk matrices comparing approaches
- **Validation evidence** — Evidence supporting or contradicting recommendations

## Cross-Domain References

Research artifacts inform decisions across all domains:

- **Design** reads `01-discover.md` for user research context during the Empathize phase
- **Marketing** reads `02-define.md` for problem framing and audience insights
- **Development** reads `04-deliver.md` for validated requirements and priorities
- **Project Management** reads `03-design.md` for solution feasibility and effort estimates
